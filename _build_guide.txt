DMV Digital Services PortalDataverse Table Build GuideMicrosoft Power Pages + DataverseState & Local Government Demo ArchitecturePurpose of This DocumentThis guide provides step-by-step instructions to build all Dataverse tables required to support the DMV Digital Services Portal demo. Follow each section in order. Tables are organized by domain: Citizen (B2C), Dealer/Partner (B2B), and Shared Reference data. Each table includes field definitions, data types, required flags, and configuration notes.SECTION 1 — Prerequisites & Environment Setup1.1  Before You BeginComplete the following before creating any Dataverse tables:Sign in to make.powerapps.com with your demo tenant admin account.Select or create an Environment dedicated to the DMV demo (recommended: 'DMV-Demo-Prod').Confirm you have the System Administrator or System Customizer security role.Enable Power Pages in the environment — go to Admin Center > Environments > [Your Env] > Add Resources.If using AI Builder or Copilot Studio features, ensure AI Builder credits are allocated to this environment.1.2  Naming ConventionsConsistency in naming is critical for Power Automate flows, Power Pages Liquid templates, and Copilot Studio topics to bind correctly.ConventionPatternExamplePublisher Prefixdmv_dmv_citizen, dmv_vehicleTable Display NameTitle Case, singularDriver LicenseSchema Namedmv_ + lowercase no spacesdmv_driverlicenseField Display NameTitle CaseLicense NumberField Schema Namedmv_ + camelCasedmv_licensenumberChoice (Option Set) Namedmv_ + descriptordmv_licensestatusRelationship NamesAuto-generated — do not override—💡 Create a custom Publisher named 'DMV' with prefix 'dmv' before creating tables. Rather than duplicating Contact, extend it with a related dmv_citizenprofile table for DMV-specific fields.💡 Do NOT delete or hide standard Contact fields (First Name, Last Name, Email). Power Pages authentication depends on them.Table SettingsSettingValueDisplay NameCitizen ProfilePlural NameCitizen ProfilesSchema Namedmv_citizenprofilePrimary Column Display NameFull NamePrimary Column Schema Namedmv_fullnameEnable AuditingYesEnable Change TrackingYesAllow Quick CreateNoFieldsField Display NameSchema NameData TypeRequiredDescription / NotesFull Namedmv_fullnameSingle Line TextYesAuto-populated from Contact.
